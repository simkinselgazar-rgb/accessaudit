Phase → Model Assignments

This table shows which model handles each phase of a review. Every assignment below is configurable in the form on this page. The routing rules live in functions/llm.py:_select_model and are driven entirely by the settings below — nothing is hard-coded.

Phase / Task Setting What it does
Phase 1 — Inventory Audit ai_local_judge_* Reads each HTML section + screenshot and reports elements the deterministic extraction missed. Uses the Image Analysis model (runs as a tool call with images attached).
Phase 2 — Visual AI Explorer ai_local_judge_* For each interactive element, compares before/after/focus screenshots and classifies the interaction type (dropdown, submenu, modal, focus_visible, etc). Image-bearing, so routed to the Image Analysis model for accuracy.
Phase 3 — Video Segment Planning ai_model (primary text) Decides which interaction sequences to record as video segments. Text-only call.
Audio Detection ai_explorer_* Watches the observation video and reports whether audio plays automatically. Uses the Explorer model because it handles video + audio natively.
Video Description (keyboard walkthrough, observation, segments) ai_video_* Watches each captured video and produces a prose description to be reused across SC checks. Videos are chunked into ≤60s clips with rolling continuity context.
Visual AI (per SC check) ai_local_judge_* Examines page screenshots for each success criterion. Image-bearing, so routed to the Image Analysis model.
Code AI (per SC check) ai_model (primary text) Reads the HTML source for each success criterion. Text-only, so uses the primary text model. Large HTML is chunked by landmark.
Judge (per SC final verdict) ai_judge_* Arbitrates between the visual/code/programmatic/AT-sim verdicts and produces the final VPAT finding text. Can use a stronger model than the primary.
Synthesis (executive summary) ai_model (primary text) Writes the executive summary across all SC results.
LLM Restructure Fallback ai_fallback_* When a tool call fails to parse after 3 retries, the raw prose is sent to this model with "convert this to a tool call" to recover the data.
Crawl Page Selection ai_model (primary text) Analyzes a site crawl and picks which pages to test.

Accuracy rule: All image-bearing calls route to the Image Analysis model (ai_local_judge_*) regardless of other settings. Smaller models like the Explorer were observed hallucinating focus indicators on byte-identical screenshots, so we always pick accuracy over speed for vision work.
